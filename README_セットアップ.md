# くらしタスク セットアップ手順

アプリ本体（index.html）は**何も設定しなくてもスマホ・PCのブラウザだけで完全に動きます**。
以下は「クラウド同期」と「改善メモの自動送信」を使うときの手順です。
どちらも1回だけの設定で、あとは自動です。

---

## ① クラウド同期（Supabase）— 入れたデータが自動でクラウドに上がる

### Supabase側（1回だけ・約5分）

1. https://supabase.com を開いてログイン（Googleでサインアップ可）
2. 「New project」を押す
3. 名前は `kurashi-task` など好きな名前 → データベースパスワードは自動生成のままでOK → リージョンは「Northeast Asia (Tokyo)」→「Create new project」
4. 左メニューの「SQL Editor」を開く
5. このフォルダの `supabase_setup.sql` の中身を全部コピーして貼り付け →「Run」
6. 左メニューの「Project Settings」→「API」を開く
7. 次の2つをメモする
   - **Project URL**（`https://xxxx.supabase.co`）
   - **anon public** のキー（`eyJ…` で始まる長い文字列）

### アプリ側

1. くらしタスクの「設定」→「クラウド同期（Supabase）」
2. さっきの **URL** と **anonキー** を貼る
3. **同期キー**に好きな合言葉を入れる（例：`kurashi-haruka-0111`。推測されにくいものに）
4. 「保存して接続」→「同期済み」と出れば完了

これで保存のたびに自動でクラウドへ上がります。
**スマホとPCで同じ3つを入れれば、同じデータが見られます。**

## ② 改善メモの自動送信（GAS → Notion改善DB ＋ Gmail）

改善メモが Notion の「🔧 改善DB」に1行ずつたまり、同時にGmailにも通知が届きます。
（モニター・販売時にお客様の声を集める仕組みの土台です）

1. https://script.google.com を開く →「新しいプロジェクト」
2. このフォルダの `gas_kaizen.gs` の中身を全部コピーして貼り付け（元からあるコードは消す）
3. 上の方の `var NOTION_TOKEN = '';` に、Notionのトークン（`ntn_…`）を入れる
   ※草刈りツールのGASで使ったものと同じでOK
4. 右上「デプロイ」→「新しいデプロイ」→ 歯車で種類「**ウェブアプリ**」を選ぶ
5. 「実行するユーザー：自分」「アクセスできるユーザー：**全員**」→「デプロイ」
6. 表示された `https://script.google.com/macros/s/…/exec` のURLをコピー
7. くらしタスクの「設定」→ 改善メモの「**改善送信用GAS URL**」欄に貼る

以降、「📮 改善DBへ送信」ボタンでNotionとGmailの両方に届きます。
（URL未設定のときは、今まで通りメールの下書きが開く方式で動きます）

## スマホで使うには

現時点ではこのファイルはPC内（`~/ClaudeCodeTools/home-hub/index.html`）にあります。
スマホで使うには、次のどちらかで「URLで開ける状態」にします（次の作業で相談）。

- Vercelにデプロイ（推奨・無料。GitHub連携で更新も楽）
- 既存の個人サーバー等に設置

※クラウド同期を設定しておけば、後からどこに置いてもデータは同期キーで引き継がれます。
